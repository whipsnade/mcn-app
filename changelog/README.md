# 变更日志目录

按日期记录本项目的功能与架构变更，供**新开的开发会话预先学习**：开始工作前先读本目录最新 2-3 篇日志，可快速建立上下文（改了什么、为什么、关键文件、验证状态、遗留事项）。

## 使用约定

- 文件命名：`YYYY-MM-DD.md`，一天一篇；同一天有多次重大变更时追加到当日文件。
- 每篇结构：背景与目标 / 主要改动（含关键文件）/ 验证结果 / 遗留事项与注意事项。
- 写作原则：写给"没有本会话记忆的后来者"，记录**决策与原因**，不只罗列 diff；已废弃的设计要注明废弃原因。
- 架构级真相仍以 `AGENTS.md` 为准；本目录是时间维度的演进记录。
