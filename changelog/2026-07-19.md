# 2026-07-19

## 背景与目标

- 完善系统管理功能：前端管理端（AdminPanel）从只读占位接入真实后端。
- 架构调整：取消 pipeline 模式，全部任务走单一 agent 迭代循环；BI 报表数据项驱动；取消 10 次工具调用上限，改为余额驱动停止。
- 新建空白会话 + brainstorm 澄清模式（superpowers brainstorming 式的一问一答澄清）。

## 主要改动

1. **管理端后端**（迁移 0017 `admin_audit_logs`）：`/api/v1/admin/*` 六个端点（用户 CRUD/积分调整/积分流水），`require_admin` 依赖，审计日志（手机号脱敏）。
2. **管理端前端**：AdminPanel 全面接入真实 API（账号列表/新增/编辑/禁用/积分调整/消耗图表），渠道 slug↔中文映射。
3. **取消 pipeline 模式**：删除 planner/batching/双 contract/候选评分/xlsx 导出与前端 BiReport/CandidateList 等组件；`kind` 固定 `"agent"`，历史表保留不写入。
4. **agent 循环强化**：BI 数据项清单（`bi_requirements.py` 8 项）注入上下文 + finish 覆盖门禁；调用次数无上限，`InsufficientPointsError` 时有余额证据仍出报告并置 `insufficient_balance` 终态。
5. **空白会话 + brainstorm**：`POST /sessions` 空 body 建空白会话（标题「新会话N」）；`brainstorm/` 模块（7 项关键字表、画像落 `filters_snapshot`、title_suggestion 提炼会话名、ready 后自动建任务）；ChatArea 选项 chips。
6. **AGENT_LOOP prompt**：指代解析规则（"相关话题/本品"的主体是会话品牌，禁止把泛指词原文当查询参数）——源于真实 case：模型曾用"相关话题"四字当关键词。

## 验证结果

- 后端 pytest 258→276 全绿；前端 Vitest 148→159 全绿。
- 实测：brainstorm 两轮澄清自动建任务、标题提炼「海底捞抖音小红书口碑」成功。
- 教训：kimi 会话关闭会杀掉后台 dev server 与在途任务（`Session closed`）；uvicorn --reload 热重载也会中断任务执行——后续后端改为无 reload 运行。

## 遗留事项与注意事项

- 积分流水链路：`mcp_calls.settlement_transaction_id → wallet_transactions`，管理端图表数据源。
- `insufficient_balance` 事件契约：`task.failed` code=insufficient_balance，前端按此识别。
