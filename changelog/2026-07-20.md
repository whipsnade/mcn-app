# 2026-07-20

## 背景与目标

- 客户新需求：用户行业属性字段 + 左侧 2x2 快捷功能（预算内达人推荐/达人活动评估/小红书爆贴/抖音爆贴），全部按行业过滤。
- 服务器模型切换：UAT 与本地从腾讯 DeepSeek 改为 Kimi（api.kimi.com/coding/v1，模型名 k3，推理模型）。
- 架构讨论后决定：agent 模式保留自研循环，不引入外部 agent 框架（pi-mono）。

## 主要改动

1. **迁移 0018**：`users.industries` JSON 多值字段（存量回填「美食」）；新表 `quick_mcp_calls`（快捷调用留痕）。
2. **快捷功能后端**（`backend/app/quick/`）：四个同步端点（达人推荐/达人详情/爆贴/评估上传），轻量计费（reserve/settle/release 加 `reference_type="quick_mcp_call"`，悬挂预留由恢复循环清扫）；行业标签解析带进程内缓存与兜底。
3. **快捷功能前端**：SessionList 2x2 按钮、KolRecommendPanel（预算滑动条）、TopPostsPanel、EvaluatePanel（xlsx/csv 上传）、KolDetailView 右栏；AdminPanel 行业多选编辑。
4. **config 放开**：删除 `TENCENT_PLAN_BASE_URL` 腾讯端点硬校验与 `TENCENT_PLAN_MODEL` Literal 限制，支持任意 OpenAI 兼容端点；UAT `.env` 切到 Kimi k3。
5. **BI 数据项驱动强化**：agent 循环 finish 覆盖门禁（8 项 BI 数据项缺失回喂、streak 上限 3）、同一工具空结果 3 次熔断、覆盖判定加 `content_signal` 内容特征兜底（voice_trend 认可按天日期序列）。
6. **AGENT_LOOP prompt 反浪费**：泛指词约束、name 禁止猜测品牌、参数硬约束表（size≤100、扁平参数、小数比例、平台标签字段差异）。

## 验证结果

- 后端 pytest 312+ 全绿；前端 Vitest 183+ 全绿。
- 达人推荐端到端实测成功（41-50 条、计费正确）；爆贴受 insight-cube 全天故障影响间歇可用。
- UAT 部署（rsync + 迁移 + Kimi k3 + 端口 40099 nginx）。
