# 2026-07-21

## 背景与目标

- 客户需求：放弃代码写死的 MCP 调用链，全部由模型决策（代码只组装 prompt + 护栏）；建立 prompt 学习日志与成功案例回放；内容选题（social-grow-content-mcp）工具接入；爆贴改 7 日榜；前端快捷入口改为 tab 页。
- 用户群体明确为线下餐饮门店运营人员，要求模型按行业画像自主选择"最准且最快"的工具路径，不写死链路。

## 主要改动

1. **MCP 决策全面模型化**（quick 模块）
   - 删除 `quick/service.py` 三条写死调用链与 `quick/tags.py`；新增 `quick/agent.py` 同步小循环：模型逐轮决策（`QuickDecision`：call_tool/finish+result），护栏保留（白名单/Schema 校验、积分预留结算、8 轮上限、连续 2 次无效报错）。
   - 爆贴/达人推荐/达人详情的响应契约对前端保持兼容。
2. **Prompt 学习日志（迁移 0019 `model_prompt_logs`）**
   - `TencentPlanAdapter.complete_json/stream_text` 统一出口记录：purpose、tags、完整 messages/response、success/failed/invalid、token 用量、耗时。请求对象加 `log_context`。
   - `model/exemplars.py`：按 purpose+tags 检索最近成功记录注入新 prompt（user JSON 的 `exemplars` 键）。
3. **内容选题服务启用**
   - `DYNAMIC_TOOL_ALLOWLIST` 注册 5 个工具（hotwords 三件套 + topic 两件套，仅小红书）；启动刷新列表加 SOCIAL_GROW_CONTENT；新 token 覆盖三服务（根 `.env`）。
   - 会话 agent 工具列表达 29 个（12 insight + 8 social-grow + 5 content + 4 bilibili）。
4. **Persona 画像注入**：`model/persona.py` 把 `users.industries` 翻译为画像（美食/餐饮 → 线下餐饮门店运营视角），注入 quick 小循环与 `AgentLoopContext.user_persona`；prompt 只给视角不给链路指令。
5. **爆贴改 7 日热榜**：`top_posts` goal 与默认时间窗（period_days=6）；前端副标题同步。
6. **证据瘦身**：`slim_quick_evidence`——sanitize_evidence 超 6000 字符硬截断会产生半个 JSON，导致模型 finish 空结果（真实 bug）；清单类载荷按字段白名单瘦身。
7. **prompt 增强**：QUICK_AGENT 加"同类失败两次改用其他工具（如热词链路）"；AGENT_LOOP 加 persona 指引。
8. **前端**：4 个快捷入口从左侧 2x2 移到中间 WorkspaceTabs（tab 栏常驻）；打开 tab 不再自动查询，改「查询/刷新」手动触发；移除「返回会话」按钮；管理面板行内积分快捷编辑（点击积分→输入/快捷键→差额调整走账本幂等）；空白会话 brand/category=null 崩溃修复（toSession 空值合并）。

## 验证结果

- 后端 pytest 333 全绿；前端 Vitest 187-190 全绿；tsc/build 通过。
- 实测：模型决策轨迹符合预期（标签匹配→直查→exemplar 引用"高效路径"）；爆贴手动查询链路走通；persona 出现在模型的 requirement_desc 中。
- 已同步 UAT（迁移 0019、新 token、29 工具启用，healthz/公网验证通过）。

## 遗留事项与注意事项

- **insight-cube 上游全天间歇性故障**（502/504/超时/熔断），失败均正确释放积分；恢复循环会中断重试在途任务（`possibly_sent_timeout` → interrupted → 300s 对账窗口）。
- 模型暂未主动选择热词链路（exemplars 全是 raw.posts 成功案例的偏差），已加 prompt 降级提示，待热词成功案例积累后应自然平衡。
- prompt 日志表增长快（每次模型调用一行，messages 可达数十 KB），暂无清理策略。
- 快捷功能单次成本浮动（20-80 积分）且耗时数分钟（模型每轮决策 10-38s + 30s 调用超时）。
- 后端以**无 --reload** 方式运行（热重载曾杀在途任务），改代码后需手动重启。
