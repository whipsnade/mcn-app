# 真实运行时单一路径设计

## 目标

开发环境保留模拟短信与微信登录；模型与 MCP 的业务执行仅允许使用真实服务，不保留模拟实现、运行时降级分支或模拟测试配置。

## 运行时边界

- 身份认证：`AUTH_MODE=mock` 保持不变，继续支持开发模拟登录。
- 大模型：固定使用腾讯 Token Plan 的 OpenAI Completions 兼容接口与 `DeepSeek-V4-Pro`。
- 数据能力：固定使用 DataTap MCP 网关及已配置令牌。
- 数据库：继续使用本地 MySQL。

## 配置契约

应用启动时必须校验以下真实配置：腾讯模型地址、模型密钥、模型名称、DataTap 令牌与 MCP 提供方。任何一项缺失、空值或不符合确认的地址/模型名称，应用应启动失败；不得回退到 Fake Model 或 Fake MCP。

真实密钥仅存放于被 Git 忽略的本地 `.env`，不写入源码、测试样例、日志或提交记录。

## 代码与测试边界

- 删除 Fake Model Adapter、Fake MCP Transport 及其运行时依赖。
- 删除依赖 Fake Model 或 Fake MCP 的测试配置、测试替身与测试文件。
- 保留不依赖模拟执行器的单元测试、集成测试和模拟登录测试；其配置也必须满足真实运行时的配置契约，但不会发起模型或 MCP 调用，除非测试明确覆盖真实集成。

## 验收标准

1. 后端启动后配置显示真实模型与真实 DataTap MCP 已启用。
2. 缺少真实模型密钥或 DataTap 令牌时，后端启动失败并给出不含敏感信息的配置错误。
3. 创建分析会话会进入真实模型规划与 DataTap 调用链路，不存在 Fake 分支。
4. 模拟登录接口仍可用于本地开发。
