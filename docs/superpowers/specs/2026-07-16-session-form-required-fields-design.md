# 新建 KOL 会话表单字段设计

日期：2026-07-16  
状态：已确认，待文档复核后实施

## 目标

将新建会话表单改为“行业筛选 + 初始分析指令”为仅有的必填输入；其余条件可按需填写。新增可选 KOL 名称搜索条件，保持现有原型的视觉语言。

## 字段规则

| 字段 | 表现 | 必填 | 持久化位置 |
|---|---|---:|---|
| 行业筛选 | 固定下拉：餐饮、茶饮、美妆、护肤 | 是 | `category` |
| 初始分析指令 / 提问 | 多行文本 | 是 | 初始用户消息 |
| KOL 名称搜索 | 文本输入 | 否 | `filters_snapshot.kol_name` |
| 品牌、活动/项目、渠道、目标人群、预算 | 现有控件 | 否 | 对应会话字段或 `filters_snapshot` |

空字符串在前端规范为 `null` 或空集合；后端不得因上述可选字段为空拒绝创建会话。会话标题优先使用品牌与活动；两者均为空时使用行业与固定标题，保证侧栏可读。

## 后端兼容

`WorkspaceCreate`、数据库列、`SessionBrief` 和 Planner 上下文需要允许品牌、目标人群和渠道为空；行业继续非空。Planner 的默认筛选编译只在对应可选输入存在时追加条件。KOL 名称搜索进入筛选快照和模型上下文，但不拼接成未校验的 MCP 参数。

## 验收

1. 只填写“美妆”和提问可成功创建、执行并恢复会话。
2. 缺少行业或提问时，前端阻止提交且后端返回稳定校验错误。
3. 填写 KOL 名称时，筛选快照与模型规划上下文可见该条件。
4. 表单延续圆角输入、紫色主按钮和现有图标样式。
